# 个人数字架构说明文档（S盘 & D盘）- 最终整合版

## 1. 目标与原则

本结构用于长期稳定管理个人数字资产，核心目标是：

1. 让新文件有唯一入口，减少决策疲劳。
2. 让正在做的事与长期归档分离，降低检索成本。
3. 让软件执行环境与数据资产物理隔离，便于维护和重装。

总原则：

- 数据归 `S:`，执行归 `D:`。
- 单一入口：所有外来文件先入 `S:\000_Inbox`。
- 进行中项目统一放 `S:\001_Workspace`；完结后再归档。
- 跨项目复用内容进入 `S:\500_Database`。

### 2.2 S盘一级目录作用（总览）

- `000_Inbox`：唯一收件箱。下载、接收、截图、临时外来文件全部先入此处。
- `001_Workspace`：本地活跃工作台（非 OneDrive）。存放最近 1-2 周高频改动文件。
- `100_Work`：公司职责与交付物。
- `200_Lab`：个人实验室归档区。用于沉淀调研、笔记与已完结项目。
- `300_Life`：现实生活刚需文件（证件、财务、健康、后勤）。
- `400_Interests`：娱乐和情绪价值内容。
- `500_Database`：跨项目复用资产与静态资料库。
- `600_Backups`：本地备份冷库。
- `700_Reserved` / `800_Reserved`：预留扩展位，当前不做强定义。
- `999_Graveyard`：几乎不再访问但暂不删除的终极冷归档。

### 2.3 S盘二级目录详细作用

#### `100_Work`

- `110_Management`：周报、月报、绩效、会议纪要、任务清单等管理性文档。
- `120_Active_Projects`：公司进行中项目主目录，按项目代号或项目名建子目录。
- `130_Reference`：行业标准、规范、竞品分析、技术手册、交接资料。
- `199_Archive_Work`：已结项项目归档，原则上只读少改。

#### `200_Lab`

- `220_Research`：技术调研材料（Datasheet 批注、对比实验、拆解记录）。
- `230_Notes_and_Ideas`：个人灵感、方法论、临时构思、项目草记（非 Obsidian 主笔记库）。
- `299_Archive_Lab`：完结/暂停的个人项目归档（从 `001_Workspace` 下沉至此）。

#### `300_Life`

- `310_Identity`：身份证件扫描件、简历、证件照、资格证明。
- `320_Finance`：账单、合同、发票、报销凭证、保修凭证。
- `330_Health`：体检报告、病历、检查结果、健康计划。
- `340_Logistics`：出行行程、家电说明书、生活后勤资料。
- `399_Archive_Life`：过期但可能需追溯的生活类历史文件。

#### `400_Interests`

- `410_Video_Audio`：电影、剧集、音乐、播客等影音内容。
- `420_Games`：游戏安装包、存档、MOD、攻略资料。
- `430_Memories`：照片、Vlog、聚会影像、纪念性素材。
- `440_Leisure_Reading`：小说、漫画、休闲杂志。
- `499_Archive_Interests`：历史娱乐资料冷归档。

#### `500_Database`

- `510_Library`：专业电子书、论文、白皮书、技术文献库。
- `520_EDA_Assets`：硬件资产库（封装库、符号库、3D 模型等）。
- `530_Snippets`：可跨项目复用的脚本、驱动、模板、工具集合。
  - `531_Scripts`：通用脚本（批处理、自动化脚本、数据处理脚本）。
  - `532_Drivers`：驱动代码片段、通信模块、硬件接口示例。
  - `533_Templates`：文档模板、测试模板、报告模板、工程模板。
  - `534_Utilities`：小型工具程序、常用命令集、可复用工具函数。
- `540_Design_Assets`：PPT 模板、字体、图标、图片素材。
- `550_Installers_Archive`：需要长期保留的安装包（大体积、稀缺、版本锁定）。
- `599_Archive_Database`：过时素材与旧版资产归档。

#### `600_Backups`

- `610_Phone_Backup`：手机导出备份（聊天导出、通讯录、相册备份）。
- `620_System_Images`：系统镜像、恢复镜像、启动盘镜像。
- `630_App_Configs`：软件配置导出、快捷键配置、个人配置快照。

## 3. D盘结构（软件盘 / 后台）

### 3.1 D盘完整目录树

```text
D:\
├─ 00_Temp_Cache
│  ├─ App_Caches
│  └─ System_Temp
├─ 10_Environments
│  ├─ Python
│  ├─ Toolchains
│  └─ Java
├─ 20_Communication_Data
│  ├─ WeChat_Files
│  └─ Tencent_Files
├─ 30_Engineering
├─ 40_Productivity
├─ 50_Media_Design
├─ 60_System_Utilities
├─ 70_Games_Entertain
└─ OneDrive
```

### 3.2 D盘一级目录作用（总览）

- `00_Temp_Cache`：纯缓存与临时文件，理论上可随时清理。
- `10_Environments`：开发环境与工具链。路径必须全英文、无空格。
- `20_Communication_Data`：微信/QQ 数据黑盒目录。不要手动删内部文件。
- `30_Engineering`：工程与硬件开发类重型软件安装目录。
- `40_Productivity`：办公效率类软件。
- `50_Media_Design`：图像、音视频处理软件。
- `60_System_Utilities`：系统辅助工具（含绿色工具）。
- `70_Games_Entertain`：游戏与娱乐平台。
- `OneDrive`：仅用于云同步用途，不承担工作台角色。

### 3.3 D盘二级目录详细作用

#### `00_Temp_Cache`

- `App_Caches`：浏览器缓存、解压临时目录、软件运行缓存、代理文件。
- `System_Temp`：系统临时文件落点（如需迁移 `%TEMP%` 可使用此目录）。

#### `10_Environments`

- `Python`：解释器、虚拟环境、pip 相关基础环境。
- `Toolchains`：编译链、SDK、交叉编译工具。
- `Java`：JDK/JRE 与相关依赖。

#### `20_Communication_Data`

- `WeChat_Files`：微信文件与会话相关本地数据目录。
- `Tencent_Files`：QQ 文件与会话相关本地数据目录。
- 说明：重要文件请通过“另存为”提取至 `S:`；此目录仅作运行数据沉淀，不手工清理明细文件。

#### `30_Engineering`

- 作用：安装 EDA、CAD、仿真、调试类重型工程软件。
- 示例：Altium Designer、AutoCAD、MATLAB、逻辑分析仪上位机。

#### `40_Productivity`

- 作用：安装文档、表格、脑图、PDF 处理等通用办公软件。
- 示例：Office、Xmind、Notion/Obsidian 桌面端、PDF 编辑器。

#### `50_Media_Design`

- 作用：安装图像、音频、视频处理软件。
- 示例：Photoshop、Premiere、OBS、播放器等。

#### `60_System_Utilities`

- 作用：系统增强与运维辅助工具（含绿色工具）。
- 示例：Everything、7-Zip、Snipaste、串口工具、网络抓包工具。

#### `70_Games_Entertain`

- 作用：游戏平台与娱乐客户端隔离区，避免与生产力软件混杂。
- 示例：Steam、Epic、音乐客户端。

## 4. OneDrive 与 Obsidian 规则（最新）

当前策略：

- `S:\001_Workspace` 不放 OneDrive，仅本地使用。
- OneDrive 只同步 Obsidian 笔记库。

建议：

- 将 Obsidian Vault 放在 `D:\OneDrive\ObsidianVault`（或等效 OneDrive 目录）。
- Obsidian 笔记允许跨领域标签和链接，不强制按 S 盘目录一一镜像。
- 笔记中引用项目实体文件时，使用链接指向 `S:` 对应路径。

边界定义：

- 文字知识与思考（Markdown） -> Obsidian Vault。
- 实体工程与大文件（代码、模型、原始数据） -> `S:`。

## 5. 关键路由规则（放置判定顺序）

当你拿到一个新文件，按以下顺序判定：

1. 是否刚进入系统且未处理？
  - 是 -> `S:\000_Inbox`
2. 是否当前正在高频处理？
  - 是 -> `S:\001_Workspace`
3. 是否属于长期归档领域？
  - 工作 -> `100_Work`
  - 个人实验调研/笔记/完结项目 -> `200_Lab`
  - 生活刚需 -> `300_Life`
  - 兴趣娱乐 -> `400_Interests`
4. 是否为跨项目复用资产？
  - 是 -> `500_Database`（例如脚本、驱动、模板）
5. 是否极少访问但暂不删除？
  - 是 -> 对应 `x99_Archive` 或 `999_Graveyard`

## 7. 文件命名规范

### 7.1 通用文档

`标题_作者(可选)_V版本号_YYYYMMDD`

示例：`主板电源方案评审_张三_V1.2_20260414.pdf`

### 7.2 时间流文件（会议/日志/记录）

`YYYYMMDD_事项名_补充说明`

示例：`20260414_周会纪要_硬件组.docx`

### 7.3 安装包命名（长期保留）

`软件名_版本_平台_备注`

示例：`Keil_MDK_v5.25_Win64_稳定版.exe`

## 8. 维护SOP（执行规范）

1. 每天或每周固定清理 `S:\000_Inbox`。
2. `S:\001_Workspace` 只保留近期活跃内容，个人项目进行中仅放这里，做完即下沉到 `S:\200_Lab\299_Archive_Lab`。
3. 安装软件后判断安装包去留：
  - 普通可重下 -> 删除。
  - 稀缺/大体积/版本锁定 -> 移入 `S:\500_Database\550_Installers_Archive`。
4. 微信/QQ 重要文件必须“另存为”到 `S:` 对应项目目录，不依赖聊天记录回溯。
5. 对于 `D:\20_Communication_Data`，只通过软件内清理工具管理空间。

---

## 9. 最终统一口径（本版本生效）

- `S:\001_Workspace` 为本地目录，不映射 OneDrive。
- OneDrive 仅用于 Obsidian 笔记同步。
- 取消 `S:\200_Lab\210_Projects`，个人项目进行中只放 `S:\001_Workspace`。
- D盘命名使用 `30_Engineering`，不使用 `30_Engineering_EDA`。
- 取消 `230_Lab_Assets`，标准脚本统一沉淀到 `530_Snippets`。
